custom-key-daemon

系统更改汇总

本页记录本地语音输入与自定义硬件键监听相关改动。个人用户名、绝对路径、精确 input event 编号和图形会话变量已使用占位符替换。

总体 仓库 服务 语音输入 护眼模式 调试

总体结果

  • 建立本地 Qwen3-ASR-GGUF 语音输入方案。
  • 将语音听写和护眼按键监听合并为 custom-key-daemon.service。
  • 旧服务保留为 legacy，但默认停用。

systemctl --user status custom-key-daemon.service --no-pager

journalctl --user -u custom-key-daemon.service -f

语音输入

按住语音键录音，松开后调用 Qwen3-ASR-GGUF 转写，并通过 fcitx5 addon 或 wtype 上屏。

{
  "voice": {
    "input_name": "keyd virtual keyboard",
    "input_device": "/dev/input/eventN",
    "trigger_code": 193,
    "trigger_mode": "hold",
    "asr_project_dir": "<ASR_PROJECT>",
    "model_dir": "<MODEL_DIR>",
    "output": {
      "backend": "fcitx5",
      "fallback": "wtype"
    },
    "copy_to_clipboard": false
  }
}

护眼模式

按下护眼键后执行本地护眼切换脚本。

{
  "eye_care": {
    "input_name": "Ideapad extra buttons",
    "input_device": "/dev/input/eventN",
    "trigger_code": 202,
    "trigger_on": "press",
    "command": "<HOME>/.local/bin/toggle-eye-care"
  }
}

调试与验证

scripts/run.sh --self-test
scripts/key-listener.py /dev/input/<eventN>
python -m py_compile src/custom_key_daemon.py scripts/key-listener.py

核心决策：通过设备名解析 event，避免重启后 /dev/input/eventN 漂移；默认关闭 wl-copy，避免剪贴板阻塞上屏。

fcitx5 addon

可选 addon 位于 contrib/fcitx5-qwen-voice/，只提供 D-Bus CommitText，不处理录音、ASR 或按键监听。

gdbus call --session \
  --dest org.qwenvoice.Fcitx5 \
  --object-path /qwenvoice \
  --method org.qwenvoice.Fcitx5.CommitText \
  'hello from qwen voice'
